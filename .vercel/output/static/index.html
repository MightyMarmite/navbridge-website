How it works

A structured workflow for every monthly cycle

NavBridge replaces the manual process of copying figures from fund manager PDFs into a master spreadsheet. Each run produces a verified, auditable update in under ten minutes — with every extraction traceable to source text.

01
Upload your master

Drop your master valuation spreadsheet and source files — PDFs, emails, capital account statements — from each underlying fund manager.

Accepts PDF & email
02
Firm detection

NavBridge identifies which manager each document belongs to via sender domain whitelist, body markers, or LLM inference — automatically, without manual tagging.

Deterministic first
03
LLM extraction

Per-firm YAML profiles encode exactly what to extract — share class names, period format, value labels, sanity ranges. Claude extracts using a forced tool schema that rejects any structural deviation.

Profile-driven
04
Citation verification

Every extracted value must carry a verbatim quote from the source document. A deterministic check confirms that exact text appears in the parsed source. Fabricated figures cannot pass this gate.

Anti-hallucination
05
Risk assessment

A rules engine surfaces HIGH and MEDIUM findings: entity mismatches, material movements, downgrade protection flags, new-row creation, and missing sender whitelists. HIGHs are auto-blocked; MEDIUMs are operator decisions.

Rules engine
06
Two-gate approval

You review the proposed updates and approve the data. NavBridge recalculates the master (Price, YTD, derived cells via LibreOffice headless) and shows you a CSV preview. You approve the CSVs. Both gates are yours.

Human in the loop
07
Download & load

Download the updated master spreadsheet and two CSV load files — Load File A and Load File B — formatted for your downstream portfolio and risk systems. Every run is logged, timestamped, and archived.

Ready for downstream

Features

Built for accuracy, not automation for its own sake

NavBridge uses AI where it adds genuine value — understanding unstructured fund documents — and deterministic verification everywhere accuracy matters.

Per-firm profiles

YAML profiles encode each manager's document structure — share class enumerations, period regex, value label mappings, hard sanity ranges. The extraction is specific to each firm, not generic.

Citation grounding

Every extracted record carries a verbatim source_quote. A deterministic check confirms that text exists in the parsed source. No quote — no record. Hallucinated figures cannot enter the workflow.

Forced tool schema

Anthropic's tool-use API enforces strict output structure. The model cannot invent fields, skip required ones, or produce malformed output. Structure-level hallucination is eliminated at the API layer.

Tiered risk findings

HIGH findings block records automatically. MEDIUM findings are surfaced for operator decision. The rules cover entity matching, material price movement, downgrade protection, new entity creation, and more.

Master recalculation

After approval, LibreOffice headless evaluates all formulas in the master workbook — Price, YTD, NAV, and derived cells populate with calculated values before the CSVs are generated.

Logged, timestamped runs

Every run is stored with status, timestamps, source count, master filename, and the full state at each pipeline stage. A complete record exists for every reconciliation cycle — ready for audit or review.
